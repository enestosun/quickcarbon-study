API Dökümantasyonu

Kullanıcıları Listeleme

“GET /users”: Bu uç nokta, tüm kullanıcıları listeler. İstek gövdesi boş olmalıdır. Başarılı bir istek durumunda, sunucu “200 OK” yanıt kodunu döndürür ve bir JSON nesnesi listesi içeren bir yanıt gövdesi döndürür. Her JSON nesnesi, bir kullanıcının tüm özelliklerini içermelidir. 

GET /users

Örnek yanıt:

[
  {
    "id": 1,
    "username": "jane.doe",
    "email": "jane.doe@example.com",
    "full_name": "Jane Doe",
    "phone": "555-555-5555",
    "role": "user"
  },
  {
    "id": 2,
    "username": "john.doe",
    "email": "john.doe@example.com",
    "full_name": "John Doe",
    "phone": "555-555-5555",
    "role": "admin"
  }
]

Kullanıcı Detayları

“GET /users/:id”: Bu uç nokta, belirtilen kullanıcının bilgilerini getirir. İstek gövdesi boş olmalıdır. Başarılı bir istek durumunda, sunucu “200 OK” yanıt kodunu döndürür ve bir JSON nesnesi içeren bir yanıt gövdesi döndürür. JSON nesnesi, belirtilen kullanıcının tüm özelliklerini içermelidir. Eğer belirtilen kimliğe sahip bir kullanıcı bulunamazsa, sunucu “404 Not Found” yanıt kodunu döndürür. 

GET /users/:id

Örnek istek:

GET /users/1

Örnek yanıt:

{
  "id": 1,
  "username": "jane.doe",
  "email": "jane.doe@example.com",
  "full_name": "Jane Doe",
  "phone": "555-555-5555",
  "role": "user"
}

Kullanıcı Oluşturma

“POST /users”: Bu uç nokta, yeni bir kullanıcı oluşturur. İstek gövdesi, oluşturulacak kullanıcının özelliklerini içeren bir JSON nesnesi olmalıdır. Başarılı bir istek durumunda, sunucu “201 Created” yanıt kodunu döndürür ve oluşturulan kullanıcının tüm özelliklerini içeren bir JSON nesnesi içeren bir yanıt gövdesi döndürür. 

POST /users

Örnek istek:

{
  "username": "jane.doe",
  "email": "jane.doe@example.com",
  "full_name": "Jane Doe",
  "password": "password",
  "phone": "555-555-5555",
  "role": "user"
}

Örnek yanıt:

{
  "id": 1,
  "username": "jane.doe",
  "email": "jane.doe@example.com",
  "full_name: "Jane Doe",
  "phone": "555-555-5555",
  "role": "user"
  }

Kullanıcı Güncelleme

PUT /users/:id Bu uç nokta, belirtilen kullanıcının özelliklerini günceller. İstek gövdesi, güncellenen kullanıcının tüm özelliklerini içeren bir JSON nesnesi olmalıdır. Başarılı bir istek durumunda, sunucu "200 OK" yanıt kodunu döndürür ve güncellenen kullanıcının özelliklerini içeren bir JSON nesnesi içeren bir yanıt gövdesi döndürür. Eğer belirtilen kimliğe sahip bir kullanıcı bulunamazsa, sunucu "404 Not Found" yanıt kodunu döndürür. 

PUT /users/:id

Örnek istek:

{
        "email": "jane.doe.new@example.com",
        "phone": "555-555-1234"
        }

Örnek yanıt:

{
        "id": 1,
        "username": "jane.doe",
        "email": "jane.doe.new@example.com",
        "full_name": "Jane Doe",
        "phone": "555-555-1234",
        "role": "user"
        }

Kullanıcı Silme

“DELETE /users/:id”: Bu uç nokta, belirtilen kimliğe sahip olan mevcut bir kullanıcıyı siler. Başarılı bir istek durumunda, sunucu “204 No Content” yanıt kodunu döndürür ve herhangi bir yanıt gövdesi döndürmez. Eğer belirtilen kimliğe sahip bir kullanıcı bulunamazsa, sunucu “404 Not Found” yanıt kodunu döndürür. 

DELETE /users/:id

Örnek istek:

DELETE /users/1

Örnek yanıt:

{}


  • 200 OK: İstek başarılı bir şekilde tamamlandı ve istenilen kaynaklar gönderildi.
  • 201 Created: İstek başarılı bir şekilde tamamlandı ve yeni bir kaynak oluşturuldu.
  • 204 No Content: İstek başarılı bir şekilde tamamlandı, ancak gönderilecek içerik yok.
  • 400 Bad Request: İstek hatalı veya eksik bilgi içeriyor.
  • 401 Unauthorized: Kimlik doğrulama başarısız oldu veya kullanıcının yetkisi yok.
  • 403 Forbidden: İstek reddedildi, çünkü kullanıcının gerekli izinleri yok.
  • 404 Not Found: İstek yapılan kaynak bulunamadı.
  • 500 Internal Server Error: Sunucu hatası nedeniyle istek tamamlanamadı.
